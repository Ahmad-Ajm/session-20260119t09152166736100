# ملخص المحادثة - session-20260119t09152166736100

1) Vision  
- المشكلة: صعوبة كتابة السيرة الذاتية (CV) وإيصالها للمهتمين.  
- الحل: إنشاء CV بسهولة عبر جمع المعلومات وتنظيمها في ملف، مع تخزينها في قاعدة بيانات للعرض والبحث.  
- القيمة: تسهيل إنشاء السيرة الذاتية ومشاركتها مع الحفاظ على الخصوصية.

2) Personas  
- أفراد يحتاجون إلى إنشاء سيرة ذاتية (مستخدم/صاحب CV).  
- زائر/مهتم يبحث ويطّلع ويصدّر السير الذاتية المتاحة.

3) Features  
A) In-scope (Now)  
- FEAT-001-"UX/UI (Baseline/System)"  
- FEAT-002-"تسجيل المستخدمين"  
- FEAT-003-"إنشاء CV من نموذج إدخال"  
- FEAT-004-"تعديل CV للمستخدم"  
- FEAT-005-"صفحة واحدة لإدخال بيانات CV"  
- FEAT-006-"رفع/إضافة صورة شخصية"  
- FEAT-007-"إدخال الاسم"  
- FEAT-008-"إدخال رقم الموبايل"  
- FEAT-009-"إدخال الخبرات"  
- FEAT-010-"إدخال التعليم"  
- FEAT-011-"عرض جميع السير الذاتية المتاحة"  
- FEAT-012-"تصدير CV للزوار مع إخفاء البيانات الحساسة"  
- FEAT-013-"بحث سريع في السير الذاتية"  
- FEAT-014-"تصفية سريعة للسير الذاتية"

B) Out-of-scope / Deferred (User said later)  
- لا يوجد مذكور.

4) Data / Content  
- بيانات CV: الاسم، رقم الموبايل، الصورة، الخبرات، التعليم.  
- بيانات المستخدم للتسجيل (لم تُحدد الحقول).  
- قائمة السير الذاتية المعروضة للزوار/المستخدمين.  
- ملف/مخرجات التصدير للسيرة الذاتية (صيغة التصدير غير محددة) مع نسخة تُخفي البيانات الحساسة (تعريف “الحساسة” غير محدد).

5) Non-Functional Requirements  
- NFR-001-"سرعة التحميل والتصفح"

6) Open / Missing Information  
- هل لكل مستخدم سيرة ذاتية واحدة أم عدة سير؟  
- ما هي البيانات الحساسة التي يجب إخفاؤها عند التصدير تحديدًا؟  
- من يستطيع رؤية “جميع السير الذاتية المتاحة” (عامة/مسجلون فقط/صلاحيات)؟  
- هل يوجد تسجيل دخول/تسجيل فقط؟ وآلية الاسترجاع/تغيير كلمة المرور؟ (غير مذكور)  
- صيغة التصدير المطلوبة (PDF/Word/صفحة قابلة للطباعة… غير مذكور).  
- تفاصيل حقول الخبرات والتعليم (نص حر أم عناصر متعددة: شركة/من-إلى/وصف… غير مذكور).  
- هل يوجد موافقة/تحكم بالخصوصية لكل CV (إظهار/إخفاء) غير مذكور.  
- اللغات المطلوبة (عربي فقط أم متعدد) غير مذكور.

الستاك (إلزامي)  
- C# + ABP + Angular
